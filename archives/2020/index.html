[나 혼자 한다 - 웹서비스편] 1. Express로 라우팅 로직 뼈대짜기

Sep 18, 2020
Develop, #1. 1인개발 - 웹서비스편

우선 지난 글에서 그린 그림을 기반으로 먼저 라우팅 로직을 구현합시다. Express를 사용하면 편하게 구현할 수 있습니다.

라우터

웹에서 라우터를 무엇이라 설명할지 애매하지만, 이 글에서 구현하고자 하는 라우팅 로직은 대부분 요청하는 URL경로에 따라 응답을 어떻게 줄지에 대한 내용을 구현합니다.

예를 들어 사용자의 요청이 GET /account/login일 경우, 우리는 로그인 사이트를 사용자에게 보내줘야 합니다. (만약 세션이 존재한다면 그냥 전 페이지로 redirect를 시켜줘야합니다)
만약 사용자의 요청이 GET /articles/142와 같을 경우에는 142번에 해당하는 게시글을 보내줘야겠조. node에서 사용자의 요청을 보고 요청 객체 아래 존재하는 URL을 파싱해도 되지만, express와 같은 프레임워크를 사용하면 훨씬 쉽고 의미적으로 깔끔한 코드를 짤 수 있습니다.

일단 이번 글에서는 해당 URL로 요청이 올 시 URL이 설정한 라우팅 로직 안에 있으면 Method와 URL을 돌려주는 것까지만 구현을 합니다. 나머지 로직들은 DB, auth와 같은 기능이 선행적으로 구현되어야 하기 때문입니다.

뼈대

폴더 구조

우선 다음과 같이 폴더를 만들어줍시다. 사실 폴더 구조는 어떻게 짜든 상관이 없으나 여러번 해봤을 때 이렇게 폴더 구조를 해놓는게 가장 편했습니다. 예상되는 URL들이 /account /archive /blog /board… 등이 있으니 해당 하위 경로마다 폴더를 만든 후에 각각 index.js와 $subpath.js를 만들어 줍시다.

필요한 폴더와 스크립트를 만들고 난 후 폴더 구조는 다음과 같습니다.

이제 각각의 js 파일에 테스트용 라우팅 로직을 넣어주면 뼈대를 완성할 수 있습니다.

변경점

20.07.19 : 좌표계 변환 식 수정 (asin -> atan2)

지금까지 한 일들은

  • HYG-database 처리
  • 간단한 스켈레톤 구현

입니다. 이제 가장 핵심이 되는 좌표계 변환을 구현해봅시다.

좌표계 변환을 알아보기 전에 간단하게 지평좌표계와 적도좌표계에 관한 내용을 다시 기억해봅시다.

천구좌표계

하늘에 있는 별들은 우리랑 상당히 먼 거리에 있어서 위치가 크게 변하지 않습니다. 하지만 우리가 좌표계를 어떻게 설정하느냐에 따라서 여러가지 좌표를 줄 수 있습니다.

예시로, 같은 점을 표현할 때 직교좌표계와 극좌표계를 이용하여 두 가지 방식으로 표현할 수 있다는 점을 생각하면 좋습니다. 두 좌표계 모두 표현할 수 있는 점의 영역이 같지만, 각각의 좌표계의 장단점이 다르고 주로 활용하는 영역이 다릅니다.

두 좌표계가 표현할 수 있는 영역이 같기 때문에, 한 좌표계의 점을 좌표계의 점으로 변환할 수 있고 일반적인 변환공식이 존재합니다.

천체관측에서 주로 활용되는 좌표계는 지평좌표계와 적도좌표계입니다. 황도좌표계와 은하좌표계는 나중에 시간이 있으면 따로 알아보도록 합시다
