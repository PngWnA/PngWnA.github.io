    지난번에는 별의 좌표정보를 사용하기 위해서 온라인에 있는 HYG-database를 얻어온 후 필요한 정보만 추출했습니다. 앞으로 해야할 일을 생각해보면
    • 해당 JSON을 브라우저에 로드
    • JSON에 들어있는 별들을 HTML의 element로(<star>로 이름지읍시다) 올리기
    • <star>의 적도좌표계 값을 현재 시간과 위경도를 이용하여 지평좌표계로 변환
    • 해당 지평좌표값들을 이용하여 window에 점들을 투사하기.

    이중에서 좌표계 변환이 가장 어려울 것 같으니 나머지를 먼저 해결합시다.

    스켈레톤 깎기

    night.html

    HTML쪽은 js와 css를 가져오는 것으로 충분합니다. 기본적인 것만 구현해놓읍시다.

    정상적으로 구현되었다면 <projection> 안에 <star>들이 들어갈겁니다.

    style.css

    css를 이용하여 document에 밤하늘을 칠하고 폰트를 바꿉시다. 그리고 별에 대한 명세를 만들어놓읍시다

    이정도로 충분할 듯 합니다.

    제가 좋아하는 폰트가 시스템에 기본으로 깔려있지 않기 때문에, 해당 폰트를
    @import url(http://cdn.jsdelivr.net/font-kopub/1.0/kopubdotum.css);
    처럼 폰트를 인터넷에서 가져와야 font-family: 'KoPub Dotum'; 처럼 폰트를 사용할 수 있습니다.

    그리고 밤하늘을 자세히 보면 지평선쪽으로 갈 수록 하늘이 밝아지는데, 해당 현상은 background: linear-gradient(rgba(16,17,32,1) 0%, rgba(12,17,80,1) 100%);처럼 linear-gradient 속성을 이용하여 구현해줍시다.

    마지막으로 star는 border-radius: 50%;를 이용하여 모양을 원으로 만들고, position: absolute속성을 줘서 자유롭게 위치를 지정할 수 있게 해주면 됩니다.

    Javascript*

    기능에 따라서 스크립트를 나눠서 관리해줍시다.

    stars.js

    stars.js는 우리가 추출한 별 데이터를 받아와서 브라우저의 메모리에 로드해주는 역할을 합니다.

    init을 이용하여 별을 브라우저의 메모리로 불러올 수 있습니다.

    처음에는 브라우저에서 직접 local 파일을 로드하는 형식으로 구현하려고 생각했었는데, 해당 방식이 생각보다 어렵고, 또 브라우저가 local 파일에 접근한다는 것 자체가 보안적으로 문제가 있을 것 같아 파일을 github에 올려놓고 해당 링크를 통하여 fetch하는 방식으로 구현했습니다.

    fetch라는 메서드가 있다보니, 생각보다 인터넷에서 가져오는 방식이 꽤 구현하기 편합니다.

    load는 로드한 JSON을 document의 element로 붙여주는 역할을 합니다.

    이제 load(init())을 호출하면 우리가 추출했던 JSON파일을 document상의 element로 올려줄 수 있습니다.

    coordinate.js

    여기서 별들의 적도좌표계 값을 지평좌표계 값으로 바꿔주면 됩니다.

    나중에 구현합시다.

    projection.js

    해당 스크립트가 별들의 좌표값을 이용하여 window에 투사해주는 역할을 합니다.

    나중에 project에서 실제로 지평좌표계를 받아 window에 투사해줄 예정이지만, 지금은 아직 구현을 못했으니, 랜덤으로 아무곳에나 뿌려주는 projectTest를 만듭시다.

    해당 메서드의 내용을 보면 별 것 없고, 진짜로 랜덤으로 <star>들의 위치를 아무데나 뿌려줍니다.

    또한 별의 크기 역시 랜덤으로 일단은 지정해놨는데, 나중에는 별의 mag값을 이용하여 별의 밝기에 따라 크기를 설정해주면 됩니다.

    debug.js

    앞에서 구현한 메서드들을 이용해 실제로 화면에 잘 나오는지 테스트해보는 스크립트입니다.

    projectTest도 저 .then안에 넣고싶지만, 비동기 문제때문인지, 다른 문제때문인지는 모르겠지만, init이 되기 전에 projectTest가 호출되서 오류가 터집니다.

    제대로 일단은 setTimeout을 이용해서 땜빵을 해놓는게 나아보입니다.

    결과

    제대로 구현되었다면 새로고침을 할 때마다 별이 랜덤한 위치에 뿌려질 것입니다.
