들어가며

지난 번에 책에 대한 짧은 후기를 남긴 적이 있다. 책에서 소개한 내용중 검색엔진에 관한 내용이 있었는데, 이참에 알아두면 좋은 내용들을 다 정리한다.

이 글은

  • robots.txt
  • sitemap
  • webmaster (google|naver)
  • google analytics
    에 관한 내용을 다룬다.

우선 들어가기 전에 지금 보고있는 이 블로그의 구성에 대해 알아보도록 하자.

Github Pages 위에서 쌩쌩 돌아가고 있는 블로그

이 블로그는 Github Pages 위에서 동작하고 있으며 Static Site Generator로 hexo를 사용하고 있다.
(실제로 웹페이지를 생성하기 위해 사용하는 작업공간은 해당 레포지토리에서 확인할 수 있다)

해당 작업공간(로컬에 있음)에서 웹페이지를 빌드하여 테스트를 한 후 github pages랑 연동된 레포지토리(PngWnA/PngWnA.github.io)에 자동으로 배포되는 형식으로 운영을 하고 있다.

비슷한 환경으로 자신의 웹사이트를 배포한다면 이 글에 있는 내용들을 빠르게 따라할 수 있을 것이라 생각한다.
(아마 무료 호스팅을 쓴다면 FTP를 이용하여 올리는 것 뺴고 다른 점이 크게 없을 것 같지만…)

robots.txt

(아주 가아끔 초보적인 CTF 문제에 등장하는) robots.txt는 사람이 읽는 파일이 아니라 검색엔진에서 색인을 생성하기 위한 봇이 읽는 파일이다.
검색엔진에 봇들은 하이퍼링크를 따라 떠다니면서 다녀간 사이트들의 색인(index)를 생성하는데, 도메인의 최상위 폴더(/)에 robots.txt가 존재하면 해당 파일을 참고해 접근해도 되는 파일들을 파악한다.

sitemap

sitemap은 해당 웹사이트의 구조를 설명하는 파일이라고 설명하면 쉽게 이해할 수 있을 것이다.
(말 그대로 site의 map이라 더 설명할 방법이 없다…)
보통 XML로 주는데, 이 블로그의 sitemap를 살펴보자.

webmaster

robots.txt와 sitemap.xml이 완성되었다. 이제 검색엔진이 우리 사이트를 보다 편하게 탐색할 수 있다!

하지만 수동적으로 검색엔진이 우리 사이트를 찾아서 열심히 탐색해주기를 바라는 것보다 더 빠른 것은 우리가 능동적으로 검색엔진에 우리 사이트를 등록하는 방법이다.

두 가지 방법을 다 사용하면 왠지 검색엔진이 우리 블로그를 더 좋아해줄 것 같다. 둘 다 해보자.

google

Google Search Console에 들어가자.

다음과 같은 화면이 우리를 반겨준다.

Google Search Console

웹사이트를 구글 서치에 등록하기 위해서는 사이트의 소유권을 증명해야 하는데, 주로 다음과 같은 방식으로 이루어진다.

  • DNS Record에 특정 내용 추가 (왼쪽)
  • 웹페이지에 특정 HTML 파일 업로드 (오른쪽)
  • 웹페이지에 특정 meta 태그 추가 (오른쪽)
  • 애널리틱스 연동 (오른쪽)

가비아나 다른 도메인 서비스를 이용해서 도메인을 구매했다면 왼쪽 방법을 사용해도 된다.
하지만 이 블로그는 Github Pages 위에 올라가 있으니 오른쪽 방법중 HTML 파일을 업로드 하는 방식을 사용하도록 하자.
(무료 호스팅의 경우에도 DNS 레코드를 변경할 수 없는 경우가 많으니 해당 방법을 따라하면 된다)

우선 오른쪽에 본인의 웹사이트의 URL을 입력하면 소유권 확인 방법이 나오는데, 첫번째 방법을 사용하기로 했으므로 첫번째 방법에 대해서 알아보자.

HTML 파일 업로드 권한을 통한 소유권 인증

특정 파일을 최상위 폴더에 올릴 것을 요구하고 있는데, 해당 html 파일을 다운받아서 올리면 끝난다.

해당 파일에 무슨 내용이 있는지 궁금할 것 같은데, 별거 없다.

인증내용 하나만 들어있는 파일이다.
(참고로 네이버도 거의 똑같은 방식을 소유권 확인에 사용한다)

해당 파일을 업로드하고 확인을 누르면 소유권 확인이 끝나고 서치 콘솔이 열린다.

Search Console 메인화면

지금 해줄 것은 sitemap 등록이니 색인 > sitemap에 들어가자

Sitemap

여기서 자신의 sitemap.xml의 위치를 등록해주면 된다.
성공하면 아래처럼 ‘성공’으로 표시가 될 것이고 오류가 있으면 ‘오류’가 표시가 될 것이다.

나머지 메뉴는 심심할때마다 클릭해보고, 일단 여기까지만 진행해놓자.

Naver

네이버도 거의 똑같은 방법으로 진행해주면 된다.
Naver Search Advisor에 들어가서 위에 있는 “웹마스터 도구”를 클릭하면 비슷한 화면이 뜰 것이다.

사이트 추가에 똑같이 본인의 웹페이지를 올린 후 HTML파일을 업로드하면 된다.

소유권 인증

구글과 같은 방법을 사용하고 있는데, HTML 파일의 내용도 같다.

HTML 파일을 올리고 소유권 확인을 끝내면 역시 비슷한 창이 뜨는데, 일단은 요청 > 사이트맵 제출에서 아까와 비슷한 방식으로 사이트맵을 제출하자.

서치어드바이저 메인

이렇게 해주면 등록이 끝났다. 다른 메뉴는 심심할때마다 둘러보자.

google analytics

앞에서 설명한 webmaster가 잘 검색되는 것을 도와주는 도구라면 analytics는 웹사이트를 이용자, 이용시간, 이용방식등을 분석하는데 도움을 주는 도구다.
우선 Google Analytics로 들어가자
자신의 사이트 이름과 URL을 입력하면 애널리틱스 계정이 생성된다.

또한, analytics용 추적 코드가 생성된다
(표시가 되지 않는다면 관리 > 추적 정보 > 추적 코드에 들어가자)

추적 코드

해당 코드를 웹사이트에 추가해주어야 한다. 모든 페이지에 해당 코드가 있어야 한다.
주로 쓰는 방법은 웹페이지의 헤더에 추가하는 방법이다.

실제 이 블로그도 header에 해당하는 부분에 해당 코드를 삽입하여 모든 페이지에서 사용되도록 사용하고 있는 중이다. 확인해보자

이렇게 넣으면 알아서 다 해준다.

실제로 이렇게 패치해서 올린 후 Google Analytics를 들어가면 Active User가 있다는 것을 확인할 수 있을 것이다.
